Hi. This is a brand new project.

Here is my big issue. I am brazilian, and I reeeeaaaaally like word games. There is this game called wordle, and it has a brazilian version called Termo (https://term.ooo/).
While Termo is cool, I got tired of it. But this new game called Wordle500 made me very very interested.
In short, wordle500 is wordle but they only tell you the number of yellow/green/red letters on every shot, and you are responsible for decoding what letters are what color. It is much harder.

The problem is that wordle500 is in english, and I want to play in brazilian portuguese. Our task today is to reproduce this exact game but with brazilian portuguese words, and by preserving Termo's color scheme.

Word500 also has 3 modes:
- Standard: no repeated letters, no JQXZ
- Standard+: no repetead letters
- Avanced: Anything goes!

in portuguese, we are mostly interested in advanced, we may do other methods later if we feel like it. Furthermore, some changes that we want that are specific to brazilian portuguese (and mirror termo):
- ç and c are considered the same letter and rendered interchangibly depending on the word. The same goes for any accent and tilde. Bocao is rendered as Bocão. vasco is rendered as vasco, chule is rendered as chulé, and braco is rendered as braço.

We should use the same colorscheme as termo. I have zero idea if you can actually see the colorscheme that termo uses, so I included a screenshot of both websites. in this folder (termo.jpg, word500.jpg)

In word500, every letter you choose is considered as gray in the keyboard down there, while in termo it stays the color the letter is (bc ofc in word500 you don't know it). We should mirror word500 in this behavior

In both websites, the letters do not turn the color immediately, there is a little animation, the block turns on its own axis. Termo's is sideways (turns using the y axis)
